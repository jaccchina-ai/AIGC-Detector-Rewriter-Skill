# Chinese-Text AI-Risk Phrase Reference (中文文本 AI 痕迹风险提示)

This reference applies ONLY when the text being revised is in Chinese (a Chinese-language thesis, a Chinese section, or a Chinese abstract). It is distinct from the Gate J "Chinese-student-English cliché" bank, which targets English text written by Chinese authors. For English thesis text, use the English banks; for Chinese text, use this one.

It is a HINT list, not an absolute ban list. Semantic accuracy and term protection always come first. All iron rules still apply: minimal edit, no regeneration, no back-translation, word count not reduced, no fabrication, protected terms/citations/data unchanged. Replace a flagged phrase by adding specific content the document already contains — never by inventing facts and never by deleting substance.

**重要：本文件各改写方向表中凡出现“删除”，一律遵守同段补偿规则——仅删除不承载实质意义的空泛措辞；若删除后该段字数下降，必须用原文已有的对象、数据、边界条件、研究场景或论证步骤补足信息量，使净字数不下降（对应 Principle 4 / Gate F），不得新增论文没有的事实。**

## A. 模板化开场句 (template openings)
| 高风险 | 问题 | 改写方向（用文中已有内容）|
|---|---|---|
| 随着……的不断发展 | 过度泛化开场 | 直接给出研究背景中的具体矛盾或问题 |
| 在当今信息化时代 / 在大数据背景下 | 空泛时代背景 | 删除，或替换为该领域的具体场景 |
| 众所周知 | 主观、无出处 | 直接陈述事实并保留原引用 |
| 近年来，越来越多的研究 | 套话开场 | 指明是哪一类研究、解决了什么 |

## B. 模板化总结句 (template closings)
| 高风险 | 问题 | 改写方向 |
|---|---|---|
| 综上所述 | 过度公式化 | 结合结果给出边界条件，或用本段已有的具体发现收束；不得只删除导致字数下降 |
| 总体而言 / 总的来说 | 空泛收束 | 用具体指标或结果收束 |
| 由此可见 / 不难看出 | 逻辑跳步 | 补上中间观察依据（文中已有的）|
| 具有重要意义 | 空泛价值判断 | 写明"对谁、在哪个场景、改善了什么" |
| 提供了新的思路 | 套话 | 说明"新"在何处（机制 / 假设 / 设置）|
| 值得注意的是 | 机械连接 | 直接写出该结论或因果 |

## C. 无依据夸张词 (unsupported intensifiers)
| 高风险 | 改写方向 |
|---|---|
| 显著提升（无数值）| 给出量化幅度，或删去"显著"（除非是统计显著且原文有 p 值）|
| 全面优化 / 全面提升 | 指明优化的具体维度 |
| 高效且鲁棒 | 给出时延 / 方差 / 噪声条件等已有证据 |
| 具有较强泛化能力 | 给出跨数据集 / 跨样本的已有证据 |
| 达到最优性能 | 改为"在给定设置下优于基线"（仅当原文确有此比较）|

注意："显著"在统计语境下（原文报告了 p 值、显著性检验、回归结果、相关分析、假设检验或表格结论）是受保护的术语，不得删改；仅当它是无依据的修辞性夸张时才处理。受保护的统计搭配还包括：显著性水平、统计显著、显著正向影响、显著负向影响、p 值、回归系数、置信区间、标准误、显著相关——这些一律不改。凡涉及"显著正向影响 / 显著负向影响 / 统计显著 / 显著相关"的句子，必须进入 Protected Elements Check 并在受保护清单中列出，防止机器误判为修辞夸张而改动。类似地，"有效"、"明显"、"提升"、"改善"、"优于" 若直接对应实验结果、模型比较、表格数据、假设检验或评价指标，也应进入 Protected Elements Check，一律不删改；只有在没有数据、引用或实验/统计支撑的纯修辞性夸张中，才可弱化或替换。

## D. 中文 AI 痕迹 / 模板感高风险重复块
- 连续 2 句都以"首先 / 其次 / 最后"开头 → 打散其中至少一处的进入方式。
- 连续段落都用"本文……提出……验证……"三连模板 → 变化其中部分段落的信息推进顺序（对应 T23）。
- 在 300–500 字内同一 4–6 字短语重复 3 次以上（如"具有重要意义"）→ 用文中已有的具体说法替换部分出现。
- 全文高频以"并且 / 同时 / 此外"作句首连接 → 按真实逻辑关系轮换，或删去冗余连接词靠语义自然过渡（对应 T2 删除优先）。

## E. 术语保护白名单（优先级高于本表）
即使看起来"模板化"，以下内容不得强行改写：专业术语（如模糊神经网络、Transformer、图卷积网络）、评估协议（如 5 折交叉验证、留一法）、工具环境（MATLAB、Python、PyTorch）、数据集 / 指标 / 符号 / 公式编号、变量名、假设编号、统计值、参考文献。本表永不覆盖术语与数据保护。

## F. 中文文本快速自检
- [ ] 是否存在"随着……发展""综上所述"等模板句？
- [ ] 是否有 4–6 字短语在短文本内重复 3 次以上？
- [ ] 是否出现无依据夸张词（显著 / 全面 / 最优），且非统计语境？
- [ ] 是否在保留术语、数据、引用的前提下完成句式变化？
- [ ] 是否保留了原论证顺序与因果链（未重排整段、未压缩）？
- [ ] 句长是否有长短变化，而非匀速？

## G. 深度 AI 痕迹模式（中文，超出统计检测范畴）

这些模式即使困惑度/突发度正常仍可能被深度检测器或人工评审识别。处理方式一律是"对已有内容做确定性替换 / 删除不承载实质意义的修饰"，**绝不**为此编造数据、研究者感受或操作细节（违反不编造铁律）。若删除模板词、膨胀词或空泛连接词后段落明显变短，必须在同一段内用原文已有的具体对象、数据、边界条件、研究场景或论证步骤补足信息量，使该段净字数不下降（对应 Principle 4 与 Gate F）；不得为补字数新增论文没有的事实、数据、感受、过程或结论。

1. **系词回避（Copula Avoidance）**：AI 不用简单的"是"，偏用复杂系动词撑场面。
   - "作为……的代表" / "扮演着……的角色" / "体现了……" → 该用"是"的地方就直接用"是"。
   - 处理：把绕弯的系动词还原为简洁判断，不改变原意。

2. **悬浮式分析（从而/进而 堆叠）**：在陈述事实后挂多个"从而/进而/由此"制造虚假分析深度。
   - 检测：一句话末尾挂了 2 个以上"从而/进而/由此/这便/由此可见"。
   - 处理：把悬挂短语独立成句，或删去不提供实质信息的部分；不得新增原文没有的因果推断。

3. **重要性膨胀（中文）**：给普通事实加"重大意义"包装。
   - 高危：至关重要 / 不可忽视 / 深远影响 / 具有重要价值 / 为……提供了重要参考 / 标志着 / 开启了新篇章。
   - 处理：删掉膨胀修饰（删后含义不变即应删），或替换为论文已有的具体表述/数据；不得新增论文没有的价值判断。

4. **模糊归因（中文）**：无引用的"有研究表明 / 学界普遍认为 / 众所周知 / 相关研究指出"。
   - 处理：要么删掉模糊归因，要么补上论文已有的具体文献编号——**不得编造文献**。

5. **公式化挑战段 / 空洞展望**："尽管取得了……成果，但仍存在一定局限性""未来研究可以进一步探讨……"。
   - 处理：用论文真实情况替换（如真实的样本/条件限制）；不得编造未做的研究计划。

6. **虚假范围（False Ranges）**："从宏观到微观""从理论到实践""涵盖了从 A 到 B 的各个方面"。
   - 处理：直接列举论文实际覆盖的范围，或删去；不得夸大覆盖面。

## H. 中文 AI 高频词清单（高频出现在 AI 化文本，但不能单独作为判定依据）

这些词在 AI 化中文文本中出现频率明显偏高，但**很多在正常论文、公文、技术文档里也会出现**，因此**不能单独作为 AI 判定依据**——必须结合重复密度、上下文功能、是否提供具体信息、以及术语保护综合判断。改写时按语境替换为更朴素的说法（同一词在不同位置用不同替代，避免制造新模板）。若该词是专业术语、统计术语、政策术语、模型名称、变量名称或论文核心概念，一律不替换；不得为避免重复而制造同义词漂移（同一概念全篇必须保持同一术语）。术语、数据、引用、统计词不在此列，受保护。

| AI 高频词 | 替换方向 |
|---|---|
| 值得注意的是 | 有意思的是 / 一个细节是 / 删除 |
| 至关重要 / 不可或缺 / 举足轻重 | 关键 / 必要 / 离不开 |
| 深入探讨 / 深度剖析 | 仔细考察 / 详细分析 / 拆解 |
| 系统性 / 全方位 / 多维度 | 全面 / 各方面 / 从不同角度 |
| 高度契合 / 紧密结合 | 吻合 / 一致 / 结合 |
| 有机融合 / 无缝衔接 | 结合 / 衔接 / 过渡 |
| 稳步提升 / 持续优化 / 全面提升 | 逐步提高 / 改进 / 整体改善 |
| 显著提升（无数值） | 明显增强 / 大幅改善（有数值则保留数值并删"显著"修辞）|
| 深远影响 / 广泛关注 | 长期影响 / 受到关注 |
| 源源不断 | 持续 / 不断 |

注意：本清单是"提示+替换方向"，不是机械全局替换。每处替换用不同的替代词，且不得为了替换而改变原意或制造网络用语/情绪化口语（违反学术语体约束与 Gate J）。

## I. 术语 / 缩写 / 符号的"重复定义"AI 腔（中文）

AI 中文学术文本的一个典型痕迹是"每段都重新定义一次"。下列约定只调整**重复出现时的写法**，不删事实、不改首次定义、不碰公式与数据：

1. **中文术语后不要每次都跟英文对译**：如"非局部雾线（haze-line）"在全文反复出现。处理：首次出现可保留一次"中文（英文）"，**后文重复时去掉括号英文**，只用中文。除非用户或期刊明确要求双语对照。
2. **缩写与全称的顺序与去重**：有通用缩写时，首次写"缩写（中文全称）"，如"DCP（暗通道先验）"；**不要**堆叠成"暗通道先验（Dark Channel Prior, DCP）"。后文只用"DCP"或"暗通道先验"其中一种，不再重复写全称括号。
3. **符号首次解释、后文不再整句重复**：同一符号（如 t(x)、Ω）首次可"其中……表示……"，后文默认读者已知，不再每次重复整句解释。
4. 这些都属于"去除重复性 AI 腔"，**不得**因此删除首次定义或改变符号/术语本身（受 E 节术语白名单保护）。

判断"首次/重复"边界时，以全文实际首次出现为准；跨章节改写时保持同一术语/缩写写法一致（不要前章 DCP、后章又改叫"暗通道方法"——那是同义词轮换，见 G2/术语保护）。

## J. 经验观察：中文 MBA 论文的风险章节分布（样本极小，仅作启发，非阈值）

来自一个 CNKI 校准检测器的真实数据（**仅 2 篇 MBA 论文，样本极小，不可当作校准阈值或权重**）的一个一致观察：AIGC 风险在**结论/对策/建议章（末章）最高**，而在实证/数据分析章（如第 3–5 章）很低甚至为 0。可能的原因是末章多为"对策建议、展望"这类概括性、模板化、少数据支撑的论述，正是 AI 高频腔的高发区；而实证章被具体数据、表格、变量、统计结果锚定，天然风险低。

实操启发（不是硬规则）：
- 在中文 MBA / 管理类论文里，优先检查**结论、对策建议、研究展望**等末章的模板腔（呼应 Front-Loaded 的"优先改高分区"，但方向相反——这类论文的高分区常在末章而非开头）。
- 末章降腔仍受全部铁律约束：对策建议不得新增论文没有的措施或数据；展望不得编造未做的研究计划（见 G5）。
- 同时记录的另一观察：这 2 篇的 CNKI 分数（0.225、0.333）明显低于该内部模型给分（0.671、0.748）——再次印证**内部估计 ≠ 外部 CNKI 实测**，任何内部判断都必须以同版本外部复测为准（见 SKILL 的 External-Risk Calibration 与"内部分数不保证外部结果"）。

因为样本只有 2 篇，本节是"先看哪里"的启发，**不修改任何评分阈值、地板或权重**；若日后有更多带真实检测分的样本，可据此再校准。
